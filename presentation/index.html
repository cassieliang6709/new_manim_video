Why LangGraph?

Control flow isn't linear — audit fail routes differently than exec fail. LangGraph separates graph topology from business logic. Adding a node doesn't touch other nodes.

Two-layer Audit

Layer 1: AST static scan — deterministic, blocks dangerous imports.
Layer 2: LLM Judge (Flash) — semantic check, 3–5× cheaper than Pro.

Debugger Node

Compresses 2000-char traceback → 2 lines:
DIAGNOSIS: root cause
FIX: concrete action
SNR ↑, token cost ↓, fix rate ↑

重点讲三个设计决策，每个约 2 分钟。 1. LangGraph vs while loop — 拓扑和逻辑分离 2. 两层 Audit — 确定性 vs 语义，互补 3. Debugger 独立节点 — 降噪，廉价任务为昂贵任务服务

03 / Engineering Challenges

"Three Problems I Didn't Expect — and How I Solved Them"

Problem 1 · Infinite Loop

LLM Judge kept returning FAIL.
No audit retry cap →
GraphRecursionError at limit 50.
User sees: blank screen.

Fix · audit_retry_count

Added audit_retry_count to GraphState. Increments on each audit fail. Routes to fallback_node at cap. Resets when new code is generated.

Lesson: Different failure types need separate counters and circuit breakers.

Problem 2 · Traceback Noise

Full tracebacks = 2000+ chars.
Real error buried in the middle.
LLM "fixed" wrong lines.
Retry success rate: low.

Fix · _compress_traceback()

Extract last 5 lines + last File "...", line N. Prepend structured signal: TIMEOUT / NO_OUTPUT / RUNTIME_ERROR. 2000 chars → 200 chars, structured.

Lesson: Root-cause classification before passing to LLM = better fix direction.

Problem 3 · NameError Guessing

NameError: name 'Write' is not defined
LLM guesses from memory.
Manim API versions differ.
Fix is imprecise or wrong.

Fix · ApiLookup

Regex extracts symbol name from error. Looks up manim_api_index.json (57 entries). Injects exact signature + runnable example into debugger hint.

Lesson: Tell LLM the right answer, not just the error.

三个问题各约 2 分钟。 Problem 1 最重要，必讲。Problem 2+3 可合并为"反馈质量的两个维度"。 每个讲完后停顿 1-2 秒，让面试官有打断提问的机会。
03b / Code Deep Dive · Error-Driven RAG

From NameError to Correct Fix — in One Lookup

1. Traceback arrives at debugger_node

NameError: name 'Write' is not defined
  File "scene_abc.py", line 14
    self.play(Write(title))

2. ApiLookup extracts symbol + looks up index

_NAMEERROR_RE = re.compile(r"name '(\w+)' is not defined")

# Matches → "Write"
# Looks up manim_api_index.json
# Returns entry with signature + example

3. Debugger hint injected into next generate call

DIAGNOSIS: Missing import for Write animation class.
FIX: Add Write to the manim import statement.

[Manim API — Write]
  Animates writing a Text or VMobject stroke by stroke.
  Signature: Write(mobject, run_time=1, **kwargs)
  Example:   self.play(Write(Text('Hello')))

Generator receives the correct answer, not just the error. NameError fix precision improves significantly.

04 / RAG System Design

Two-Tier Retrieval: Generation + Debug

Tier 1 — Few-shot at Generation Time

Source: runs.json (successful past scenes)
Model: all-MiniLM-L6-v2 (sentence-transformers)
Strategy: Cosine similarity over prompt embeddings → top-2 examples injected as few-shot
Cold start: Returns [] gracefully → zero-shot fallback

When to use: first attempt only — retry path already has specific error feedback
+

Tier 2 — Error-Driven at Debug Time

Source: manim_api_index.json (57 hand-curated entries)
Model: Regex — no ML needed, deterministic
Strategy: Extract symbol from NameError/AttributeError → exact doc lookup
Signal-to-noise: Only injects docs for the exact failing symbol

Design principle: error-driven fine-grained retrieval > broad retrieval at generation time

Future: Full Manim v0.18 docs vectorized by class/method as chunk unit → vector DB → two-level lookup (static index first, vector DB on miss)

What's Next
如果面试官问"你的 RAG 和普通 RAG 有什么区别"—— 关键是"error-driven"：我知道具体哪个 API 出问题了，所以可以精确查，而不是泛检索。 两种策略互补：Tier 1 提升生成质量，Tier 2 提升 debug 精度。

06 / Q&A Prep

Common Follow-up Questions

"Why LangGraph over a while loop?"

Control flow isn't linear — different failures route differently. Graph topology + business logic stay separate. Adding a node doesn't touch others.

"Can LLM Judge give false positives?"

Yes — but misclassification costs one extra retry, not a wrong video. Execution layer still catches runtime errors. Soft constraint, not hard gate.

"Why Gemini not GPT-4?"

Free API quota for development; architecture is model-agnostic — swap to GPT-4 in one line. The engineering decisions don't change.

"Docker cold start latency?"

First pull is slow; subsequent runs use local cache. Production fix: pre-warmed container pool. MVP tradeoff: isolation over speed.

"Concurrent write to runs.json?"

Not handled in MVP — single-user scenario. Next: SQLite with WAL mode for native concurrent write support.
