AI Animation System Demo

From plain English to rendered Manim video.

`new_manim_video` is an AI pipeline that turns natural-language prompts into math and CS animations. It does not stop at generated code: it retrieves context, audits the output, runs rendering inside a sandbox, retries after failure, and returns a real video artifact.

Why this project matters

The interesting part is not just prompting an LLM. The project is about making an unreliable generation loop behave like an actual system that can ship outputs.

Challenge

Prompts are vague

Users describe intent in natural language, but Manim needs exact scene structure, timing, positioning, and API-correct animation logic.

Engineering

Execution is the truth

A generated scene only counts if it survives auditing, runs in isolation, and renders an output video instead of crashing with a traceback.

Outcome

Reliable demo loop

The system combines retrieval, sandboxing, targeted debugging, and bounded retries so the final user experience is stronger than a single-shot prompt.

System architecture

The repo is structured as a workflow engine with separate responsibilities for generation, review, execution, retrieval, and output delivery.

Pipeline

Multi-step generation flow

01
Retrieve Pull similar successful runs and targeted Manim API references to ground the next generation.
02
Generate Use an LLM to write scene code from the prompt, style, and recovered context.
03
Audit Run AST security checks and semantic review before any rendering happens.
04
Execute Render inside Docker so unsafe or broken code stays contained.
05
Recover Compress errors, inject better hints, retry generation, and fall back safely when needed.
Tech Stack

Core building blocks

  • Python application layer with reusable service entry points.
  • LangGraph orchestration for multi-stage execution and retry routing.
  • Gemini models for generation, judging, and diagnosis.
  • Docker sandbox for rendering Manim scenes in isolation.
  • Sentence Transformers retrieval over prior successful runs.
  • Optional Google Drive upload and MCP integration.

The full implementation lives in the repository, including `app.py`, `service_api.py`, `orchestrator.py`, `executor.py`, and the template library used for sample scenes.
